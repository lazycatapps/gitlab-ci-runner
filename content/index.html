Welcome to GitLab CI Runner Manager

GitLab CI Runner Manager is a web management interface designed for GitLab Runner, allowing you to easily deploy and manage multiple GitLab CI/CD Runners on the Lazycat platform.

GitLab Runner is a core component of GitLab CI/CD, responsible for executing continuous integration and continuous delivery tasks. Through the web interface provided by this tool, you can easily register, start, stop Runners, view running status and logs without complex command-line operations.

Important Notice: The current version only supports GitLab Runner in shell mode. Other executor modes such as Docker executor and Kubernetes executor are not yet supported. Stay tuned for future updates!

Quick Start

  1. Deploy Application on Lazycat Platform

    Visit Lazycat Platform, search and deploy "GitLab CI Runner Manager" with one click.

  2. Access Web Interface

    After successful deployment, access the web management interface through the application's domain.

  3. Register Runner

    Click the "Register Runner" button and enter your GitLab instance URL and Runner Token.

    • GitLab URL: Your GitLab instance address, e.g., https://gitlab.com
    • Runner Token: Obtain from Settings â†’ CI/CD â†’ Runners in your GitLab project or group
  4. Manage Runner

    After successful registration, the Runner will start automatically. You can view status, restart, view logs, or delete Runners through the interface.

API Endpoints

This application provides RESTful API interfaces for automation management:

Method Path Description
GET /api/version Get version information
POST /api/runners/register Register new Runner
GET /api/runners Get all Runners list and status
POST /api/runners/delete Delete specified Runner
POST /api/runners/restart Restart specified Runner
GET /api/runners/logs?name=<runner-name> Get Runner logs

Frequently Asked Questions

How to obtain Runner Token?

Steps to get Runner Token:

  • Project Level: Go to GitLab Project â†’ Settings â†’ CI/CD â†’ Runners â†’ Expand â†’ Copy Registration token
  • Group Level: Go to GitLab Group â†’ Settings â†’ CI/CD â†’ Runners â†’ Expand â†’ Copy Registration token
  • Instance Level: (Admin permission required) Admin Area â†’ Overview â†’ Runners â†’ Register an instance runner

What to do if registration fails?

Please check the following:

  • Is the GitLab URL correct and accessible from within the container?
  • Is the Runner Token valid and not expired?
  • Is the network connection normal?
  • Check application logs for detailed error information

Why does the Runner show as stopped?

Possible reasons:

  • Runner process crashed or exited abnormally
  • Insufficient system resources
  • GitLab server connection interrupted

You can try restarting the Runner or check logs to understand the specific reason.

How to view detailed logs?

There are two ways to view logs:

  • Web Interface: Click the "View Logs" button in the Runner list to view the last 1000 lines of logs
  • Host Machine: Access the /data/appvar/cloud.lazycat.app.xiao.gitlab-ci-runner/configs/logs/ directory to view complete log files

Which GitLab versions are supported?

This tool uses the latest version of GitLab Runner and theoretically supports all mainstream GitLab versions (including GitLab CE, GitLab EE, and GitLab.com).

Note: GitLab version 12.0 and above is recommended for best compatibility.
